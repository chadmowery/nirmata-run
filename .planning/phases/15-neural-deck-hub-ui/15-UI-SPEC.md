# Phase 15 — UI Design Contract

> Visual and interaction contract for the Neural Deck Hub — the between-run management interface. Covers Shell inspection, equipment management, Blueprint workshop, Vault management, and run mode launcher with Ritual ceremony. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (CSS modules) |
| Preset | not applicable |
| Component library | none (custom React components) |
| Icon library | lucide-react (already installed) |
| Font | Condensed sans-serif (system fallback: "Arial Narrow", "Helvetica Condensed", sans-serif) |

**Source:** Inherits from Phase 12/14 UI-SPEC. Project uses CSS modules with CSS custom properties in globals.css. No design system framework. lucide-react already in package.json for iconography.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, slot box internal padding |
| sm | 8px | Compact element spacing, tab bar item gaps, item card row gaps |
| md | 16px | Default element spacing, panel padding, tooltip padding |
| lg | 24px | Section padding, Shell stat card padding, tab content padding |
| xl | 32px | Layout gaps between panels, Shell carousel arrow spacing |
| 2xl | 48px | Major section breaks, Ritual overlay margins |
| 3xl | 64px | Page-level spacing (Hub outer margins) |

Exceptions: none

**Source:** Inherits from Phase 12/14 UI-SPEC. Standard 8-point scale established in globals.css `--space-*` tokens. Phase 15 introduces no new density requirements — 5-tab layout fits within existing scale.

**Source:** CONTEXT.md D-17 (cyan compatible / pink incompatible drag glow), D-13 (Legacy desaturated styling), D-24 (mode card color coding). VIS-01 palette. Phase 14 UI-SPEC color contract extended.

## Focal Points & Visual Hierarchy

### SHELL Tab
- **Primary focal point:** Shell stat card centered — largest element, stat bars draw the eye
- **Secondary:** Port slot boxes below stats — visual equipment status at a glance
- **Tertiary:** Carousel arrows at edges — navigation affordance

### LOADOUT Tab
- **Primary focal point:** Shell slot boxes (left panel) — the equipment "rack" metaphor
- **Secondary:** Stash item list (right panel) — source inventory
- **Tertiary:** Filter tabs above stash list — type filtering

### WORKSHOP Tab
- **Primary focal point:** Blueprint compilation area — two columns with compile button between
- **Secondary:** Software shop below — purchasable items
- **Tertiary:** Shell upgrades at bottom — Flux spending

### VAULT Tab
- **Primary focal point:** Overflow banner (if present) — high-urgency, pink-themed, above grid
- **Secondary (default):** 30-slot grid centered — the inventory at a glance
- **Tertiary:** Slot counter in corner — capacity awareness

### INITIALIZE Tab
- **Primary focal point:** Three run mode cards across center — large, color-coded
- **Secondary:** Rules panel below selected card — expanded detail
- **Tertiary:** Leaderboard below mode cards — score context

- Font: body 14px, monospace override (`'Courier New'`), green `--text-color`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Tab order: TabBar tabs → active tab content → interactive elements. Arrow keys navigate Shell carousel and mode cards. Enter/Space activates. Escape closes overlays. |
| Focus indicators | 2px cyan outline on all focusable elements (`outline: 2px solid var(--vibrant-cyan); outline-offset: 2px`) |
| Drag-and-drop alt | Although D-14 specifies drag-only, keyboard users can still use Tab to focus a stash item, Enter to "pick up", Tab to focus a slot, Enter to "place" — programmatic drag simulation |
| Screen reader labels | `aria-label` on all tabs, slot boxes, item cards. `aria-live="polite"` on wallet display, overflow count, compile status. `role="dialog"` on Ritual overlay with `aria-modal="true"` |
| Color contrast | All text on black exceeds WCAG AAA: white (#FFFFFF) = 21:1, cyan (#00F0FF) = 14:1, pink (#FF0055) = 10:1 |
| Animation opt-out | `@media (prefers-reduced-motion: reduce)` — disable carousel transitions, compile slide, boot sequence typewriter. Instant state changes instead. |

**Source:** Inherits from Phase 14 UI-SPEC accessibility contract. Programmatic drag simulation added for keyboard accessibility despite D-14 drag-only decision.
