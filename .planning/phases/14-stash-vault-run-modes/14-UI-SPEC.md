## Interaction States

### VaultOverflowModal (Phase 15 implementation)

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Mount | Overlay fades in 200ms, pink border glow, game blocked | profile.overflow.length > 0 on run launch |
| Item hover | Cyan border highlight, show item details tooltip | Mouse over item card |
| Discard hover | Pink button glow, show confirmation tooltip | Mouse over Discard button |
| Sell hover | Orange button glow, show Scrap amount tooltip | Mouse over Sell button |
| Unmount | Overlay fades out 200ms, unblock run launch | profile.overflow.length === 0 |

### RunModeSelector (Phase 15 implementation)

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Available | Mode card interactive, cyan/pink badge, hover glow | Attempt available for mode |
| Unavailable (Attempt used) | Mode card dimmed, red "EXHAUSTED" badge, countdown timer | Attempt already used this period |
| Unavailable (Overflow) | All mode cards dimmed, red overlay message "RESOLVE_VAULT_OVERFLOW" | profile.overflow.length > 0 |
| Selection | Selected mode card highlighted, rules display expands, launch button enabled | Mode card clicked |

### LeaderboardPanel (Phase 15 implementation)

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Loading | Skeleton rows with shimmer animation | API request pending |
| Populated | Ranked list, top-3 cyan gradient, current player pink highlight | API response received |
| Empty | Empty state message, "NO_RUNS_RECORDED" | entries.length === 0 |
| Entry hover | Expand to show detailed stats (depth, kills, loot, turns) | Mouse over leaderboard entry |

**Source:** CONTEXT.md D-07 (overflow blocks launch), D-12 (run mode rules), D-15/D-17 (attempt tracking), D-19 (leaderboard storage). Phase 12 established overlay fade timing (200ms), button hover glow pattern, and modal interaction patterns.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Tab order: mode cards → launch button. Arrow keys navigate between mode cards. Enter/Space selects mode. Escape closes overflow modal (blocks run until resolved). |
| Focus indicators | 2px cyan outline on focused card/button (outline: 2px solid #00F0FF) |
| Screen reader labels | aria-label on all mode cards ("Simulation mode, unlimited attempts, full loot"), aria-live="polite" on attempt status countdown, role="dialog" on VaultOverflowModal with aria-labelledby, aria-describedby for overflow count |
| Color contrast | All text on black background meets WCAG AAA (21:1 for white, 14:1 for cyan, 10:1 for pink — all exceed 7:1 AAA threshold). Badges use background color fill with white text for sufficient contrast. |
| Animation opt-out | Respect prefers-reduced-motion: disable overlay fade transitions, instant modal mount, static leaderboard entry expansion |

**Source:** Inherits from Phase 12 UI-SPEC accessibility requirements. Phase 14 introduces no new accessibility challenges — all patterns follow established modal/overlay/button accessibility contracts.

## API Contract

Phase 14 introduces the following API endpoints for Phase 15 UI consumption:

### Vault Overflow Management

| Endpoint | Method | Purpose | Request | Response |
|----------|--------|---------|---------|----------|
| `/api/vault/overflow` | GET | List overflow items | `{ sessionId }` | `{ overflow: VaultItem[] }` |
| `/api/vault/discard` | POST | Discard overflow item | `{ sessionId, entityId }` | `{ success: boolean }` |
| `/api/vault/sell` | POST | Sell overflow item for Scrap | `{ sessionId, entityId }` | `{ success: boolean, scrapGained: number }` |

### Run Mode Management

| Endpoint | Method | Purpose | Request | Response |
|----------|--------|---------|---------|----------|
| `/api/run-mode/available` | GET | Get available modes, attempt status | `{ sessionId }` | `{ modes: { simulation: true, daily: { available: boolean, resetIn: number }, weekly: { available: boolean, resetIn: number } } }` |
| `/api/run-mode/launch` | POST | Launch run with mode + seed | `{ sessionId, mode: RunMode }` | `{ success: boolean, seed: string, runMode: RunModeConfig }` |

### Leaderboard

| Endpoint | Method | Purpose | Request | Response |
|----------|--------|---------|---------|----------|
| `/api/leaderboard/submit` | POST | Submit run score (auto, server-validated) | `{ sessionId, runStats: RunStats }` | `{ success: boolean, score: number, rank: number }` |
| `/api/leaderboard/daily` | GET | Get daily leaderboard | `{ date: string }` | `{ period: string, seed: string, entries: LeaderboardEntry[] }` |
| `/api/leaderboard/weekly` | GET | Get weekly leaderboard | `{ week: string }` | `{ period: string, seed: string, entries: LeaderboardEntry[] }` |

### Seed Rotation (Admin)

| Endpoint | Method | Purpose | Request | Response |
|----------|--------|---------|---------|----------|
| `/api/admin/seed-rotation` | GET | Get current seed rotation | `{}` | `{ daily: SeedInfo, weekly: SeedInfo }` |
| `/api/admin/seed-rotation` | POST | Manually rotate seed | `{ period: 'daily' | 'weekly' }` | `{ success: boolean, newSeed: string }` |
| `/api/admin/reset-attempts` | POST | Reset attempt tracking for period | `{ period: 'daily' | 'weekly' }` | `{ success: boolean, affectedProfiles: number }` |

**Source:** CONTEXT.md D-08 (overflow management API-only), D-14 (Ritual API-only), D-18 (seed rotation admin API), D-21 (score submission server-validated). RESEARCH.md Pattern 3, 4, 5 (extraction-to-vault, leaderboard, seed rotation).
