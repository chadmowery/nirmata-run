<task type="auto" tdd="true">
  <name>Task 1: Entity builder and registry — mixin resolution, Zod validation, template storage</name>
  <files>src/engine/entity/types.ts, src/engine/entity/builder.ts, src/engine/entity/registry.ts, src/engine/entity/builder.test.ts, src/engine/entity/registry.test.ts</files>
  <behavior>
    - RawTemplate has name, components map, optional mixins array, optional overrides block
    - resolveMixins resolves mixin references recursively (max depth 3), merging components
    - Conflicting components across mixins throw with descriptive error (per CONTEXT.md locked decision)
    - Overrides are deep-merged per-component onto resolved components
    - buildEntity validates each component against its ComponentDef's Zod schema via safeParse
    - buildEntity throws on unknown component key with message including template name and key
    - buildEntity throws on Zod validation failure with message including template name, component key, and Zod error
    - Circular mixin references are detected and throw with the circular chain listed
    - EntityRegistry.register(template) stores by name; registry.get(name) retrieves
    - EntityRegistry.has(name) returns boolean
    - EntityRegistry.register throws on duplicate name
  </behavior>
  <action>
    1. Create src/engine/entity/types.ts:
       - `RawTemplate` interface: `{ name: string; components: Record<string, unknown>; mixins?: string[]; overrides?: Record<string, Record<string, unknown>> }`
       - `ComponentRegistry` interface: `{ get(key: string): ComponentDef<any> | undefined; has(key: string): boolean; }` — used by builder to look up schemas for validation

    2. Create src/engine/entity/registry.ts:
       - `EntityRegistry` class:
         - Private `templates: Map<string, RawTemplate>()`
         - `register(template: RawTemplate)` — stores by template.name; throws if name already registered: `"Template '${name}' is already registered"`
         - `get(name: string): RawTemplate | undefined`
         - `has(name: string): boolean`
         - `getAll(): RawTemplate[]` — returns all registered templates (useful for validation sweeps)

    3. Create src/engine/entity/builder.ts:
       - `resolveMixins(template, templateRegistry, maxDepth=3, seen=[])` function:
         - If template has no mixins, return template.components as-is
         - For each mixin name, look up from templateRegistry (EntityRegistry)
         - Track `seen` set for circular detection — throw: `"Circular mixin reference: ${seen.join(' → ')} → ${mixinName}"`
         - Check depth — throw if exceeded: `"Mixin depth exceeded (max ${maxDepth}) for template '${template.name}'"`
         - Merge mixin components in order. If same component key appears in multiple mixins, throw: `"Template '${template.name}': component '${key}' defined in multiple mixins [${sources}]. Use 'overrides' to resolve."`
         - Merge template's own components on top of mixin components
         - Apply overrides block via deep merge per-component
         - Return resolved components map
       - `buildEntity(world, resolvedComponents, componentRegistry)` function:
         - Create entity via world.createEntity()
         - For each component key in resolved components:
           - Look up ComponentDef from componentRegistry — throw if unknown: `"Template: unknown component '${key}'"`
           - Validate data via def.schema.safeParse(data) — throw if invalid: `"Template, component '${key}': ${zodError.message}"`
           - Call world.addComponent(entity, def, validatedData)
         - Return EntityId

    4. Create src/engine/entity/builder.test.ts:
       - **Template resolution tests:**
         - Simple template (no mixins) resolves its own components
         - Template with one mixin inherits mixin components
         - Template's own components override mixin components with same key
         - Overrides block deep-merges specific fields (e.g., override health.max but keep health.current from mixin)
         - Nested mixins resolve recursively (A uses B which uses C)
       - **Error handling tests:**
         - Circular mixin throws with chain listed
         - Mixin depth > 3 throws
         - Conflicting component across two mixins throws with both mixin names
         - Unknown component key throws with template name and key
         - Zod validation failure throws with template name, component key, and error details
       - **Build tests:**
         - buildEntity creates entity in World with all components attached
         - Built entity's components are accessible via world.getComponent with correct types
         - buildEntity returns valid EntityId

    5. Create src/engine/entity/registry.test.ts:
       - register + get round-trips template
       - has returns true for registered, false for unregistered
       - get returns undefined for unknown template
       - register duplicate name throws
       - getAll returns all registered templates
  </action>
  <verify>
    <automated>npx vitest run src/engine/entity/builder.test.ts src/engine/entity/registry.test.ts --reporter=verbose</automated>
  </verify>
  <done>Entity builder resolves mixins (recursively, with cycle/depth guards), validates with Zod, and stamps entities into World. Registry stores and retrieves templates by name. All error cases produce clear messages. Tests pass.</done>
