<task type="auto" tdd="true">
  <name>Task 1: 2D Grid data structure with spatial indexing</name>
  <files>src/engine/grid/types.ts, src/engine/grid/grid.ts, src/engine/grid/grid.test.ts</files>
  <behavior>
    - Grid(width, height) creates a grid with width*height tiles, all initialized with default values
    - grid.getTile(x, y) returns the Tile at coordinates, or undefined if out of bounds
    - grid.setTile(x, y, tileData) updates tile properties
    - grid.isWalkable(x, y) returns the tile's walkable flag (false for out-of-bounds)
    - grid.isTransparent(x, y) returns the tile's transparent flag (false for out-of-bounds)
    - grid.getEntitiesAt(x, y) returns Set of EntityIds on that tile's entity layer
    - grid.getItemsAt(x, y) returns Set of EntityIds on that tile's item layer
    - grid.addEntity(entityId, x, y) adds entity to tile's entity set
    - grid.removeEntity(entityId, x, y) removes entity from tile's entity set
    - grid.moveEntity(entityId, fromX, fromY, toX, toY) removes from old tile, adds to new
    - grid.addItem(entityId, x, y) adds to tile's item set
    - grid.removeItem(entityId, x, y) removes from tile's item set
    - grid.inBounds(x, y) returns true if coordinates are valid
    - Flat array storage with y * width + x indexing
  </behavior>
  <action>
    1. Create src/engine/grid/types.ts:
       - `Tile` interface: `{ terrain: string; walkable: boolean; transparent: boolean; entities: Set<EntityId>; items: Set<EntityId> }`
       - `GridConfig` interface: `{ width: number; height: number }`
       - Default tile factory function: `createDefaultTile()` returns `{ terrain: 'floor', walkable: true, transparent: true, entities: new Set(), items: new Set() }`

    2. Create src/engine/grid/grid.ts:
       - Grid class with constructor(width, height) that creates flat `Tile[]` array of size width*height
       - Each tile initialized via createDefaultTile()
       - Private `toIndex(x, y)` → `y * width + x`
       - `inBounds(x, y)` → checks 0 <= x < width && 0 <= y < height
       - `getTile(x, y)` → bounds check, return tile or undefined
       - `setTile(x, y, partial)` → merge partial tile data onto existing tile (Object.assign style)
       - `isWalkable(x, y)` → `getTile(x,y)?.walkable ?? false`
       - `isTransparent(x, y)` → `getTile(x,y)?.transparent ?? false`
       - `getEntitiesAt(x, y)` → `getTile(x,y)?.entities ?? new Set()`
       - `getItemsAt(x, y)` → `getTile(x,y)?.items ?? new Set()`
       - `addEntity(entityId, x, y)` → add to tile's entity set
       - `removeEntity(entityId, x, y)` → delete from tile's entity set
       - `moveEntity(entityId, fromX, fromY, toX, toY)` → removeEntity + addEntity
       - `addItem(entityId, x, y)` / `removeItem(entityId, x, y)` → same pattern for items
       - Expose `readonly width` and `readonly height`

    3. Create src/engine/grid/grid.test.ts:
       - **Storage tests:**
         - Grid constructor creates grid with correct dimensions
         - getTile returns tile for valid coordinates
         - getTile returns undefined for out-of-bounds coordinates
         - setTile updates tile properties (e.g., set walkable to false)
         - inBounds returns true/false correctly
       - **Walkability tests:**
         - Default tiles are walkable
         - isWalkable returns false after setting walkable=false
         - isWalkable returns false for out-of-bounds
       - **Entity layer tests:**
         - addEntity + getEntitiesAt returns entity in set
         - removeEntity removes entity from set
         - moveEntity transfers entity between tiles
         - Multiple entities can occupy same tile (multi-occupancy)
         - getEntitiesAt returns empty set for empty tile
       - **Item layer tests:**
         - addItem + getItemsAt returns item in set
         - removeItem removes item from set
         - Items and entities are independent layers on same tile
       - **Multi-layer tests:**
         - Tile has independent terrain, entity, and item data
         - Modifying one layer doesn't affect others
  </action>
  <verify>
    <automated>npx vitest run src/engine/grid/grid.test.ts --reporter=verbose</automated>
  </verify>
  <done>Grid class provides 2D tile storage with flat array, walkability queries, entity/item spatial lookups with multi-occupancy, and multi-layer tile data. All tests pass.</done>
</task>

    - Game components file does NOT import from engine internals (only from engine public API / types)
