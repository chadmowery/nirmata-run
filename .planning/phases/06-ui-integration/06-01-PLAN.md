<objective>
Establish the reactive state bridge between the vanilla engine and React.

Purpose: To create a decoupled bridge where the engine's internal events drive a reactive state store (Zustand) that the React UI can safely consume without direct dependencies on the engine's internal World state or PixiJS objects.
Output: A functional vanilla Zustand store and a sync-bridge that populates it via engine events.
</objective>

<task type="auto" tdd="true">
  <name>Task 3: Implement Sync Bridge</name>
  <files>src/game/ui/sync-bridge.ts</files>
  <behavior>
    - Bridge attaches listeners to engine `EventBus`
    - Engine events correctly trigger Zustand store updates
    - Listens for `DAMAGE_DEALT`, `HEALED`, `XP_GAINED`, `ENTITY_DIED`, `MESSAGE_EMITTED`, and `STATE_TRANSITION`
  </behavior>
  <action>
    1. Create `syncEngineToStore(context: GameContext)` function.
    2. Register listeners on `context.eventBus`:
       - `DAMAGE_DEALT` / `HEALED` → Update `player` stats if target is player.
       - `XP_GAINED` → Update `player.xp`.
       - `MESSAGE_EMITTED` → Call `gameStore.getState().addMessage`.
       - `STATE_TRANSITION` → Update `gameStatus`.
    3. Ensure bridge handles initial state sync upon attachment.
  </action>
  <verify>
    <automated>npx vitest src/game/ui/__tests__/store.test.ts</automated>
  </verify>
  <done>Sync bridge correctly maps engine events to UI state updates</done>
</task>
