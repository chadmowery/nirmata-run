<task type="auto" tdd="true">
  <name>Task 1: Generic StateMachine class</name>
  <files>src/engine/state-machine/types.ts, src/engine/state-machine/state-machine.ts, src/engine/state-machine/state-machine.test.ts</files>
  <behavior>
    - StateMachine(initialState, stateConfigs, transitionTable) creates FSM starting at initialState
    - transition(to) returns true and calls onExit/onEnter hooks for valid transitions
    - transition(to) returns false and does NOT change state for invalid transitions
    - getCurrentState() returns the current state
    - canTransitionTo(to) returns whether a transition from current state to 'to' is valid
    - onEnter hook receives optional context object
    - onExit hook receives optional context object
    - Transitioning to the same state the FSM is already in returns false (self-transitions not allowed unless explicitly listed)
  </behavior>
  <action>
    1. Create src/engine/state-machine/types.ts:
       - `StateConfig<TState extends string, TContext = void>` interface with optional `onEnter(context: TContext) => void` and `onExit(context: TContext) => void`
       - Export the interface

    2. Create src/engine/state-machine/state-machine.ts:
       - Generic `StateMachine<TState extends string, TContext = void>` class
       - Constructor takes: `initialState: TState`, `stateConfigs: Record<TState, StateConfig<TState, TContext>>`, `transitionTable: Array<[TState, TState]>`, optional `context: TContext`
       - Build `Map<TState, Set<TState>>` from transition table for O(1) lookup
       - `transition(to: TState): boolean` — check allowed set, call onExit on old state, update state, call onEnter on new state, return true. If not allowed, return false without side effects
       - `getCurrentState(): TState`
       - `canTransitionTo(to: TState): boolean`
       - `setContext(context: TContext): void` — allows updating context after construction (needed for circular references in setup.ts)

    3. Write tests (RED then GREEN):
       - Valid transition changes state and returns true
       - Invalid transition returns false and state unchanged
       - onEnter called with context on valid transition
       - onExit called with context on valid transition
       - Hook execution order: onExit fires before onEnter
       - Multiple transitions in sequence work correctly
       - canTransitionTo returns correct boolean
       - Self-transition to current state returns false (unless explicitly in table)
  </action>
  <verify>
    <automated>npx vitest run src/engine/state-machine/state-machine.test.ts --reporter=verbose</automated>
  </verify>
  <done>StateMachine class passes all tests: valid/invalid transitions, enter/exit hooks with correct ordering, context passing, canTransitionTo queries</done>
</task>
