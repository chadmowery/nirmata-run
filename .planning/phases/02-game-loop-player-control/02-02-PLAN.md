  <name>Task 2: TurnManager implementation</name>
  <files>src/engine/turn/turn-manager.ts, src/engine/turn/turn-manager.test.ts</files>
  <behavior>
    - submitAction(action) processes player action only when phase is AWAIT_INPUT (TURN-06)
    - submitAction triggers complete cycle: player action → pre-turn → enemy turns → post-turn → energy ticks until player ready → AWAIT_INPUT
    - canAcceptInput() returns true only at AWAIT_INPUT phase (TURN-02, TURN-06)
    - getReadyActors() returns entities with energy >= threshold, sorted: player first, then entity ID ascending (TURN-03)
    - Dead entities (health.current <= 0) are skipped in enemy processing (TURN-04)
    - Energy tick: all living actors gain energy += speed (TURN-05)
    - Sub-tick loop: if player below threshold after tick, process enemy turns and tick again (TURN-05)
    - Max sub-tick guard of 100 prevents infinite loop from degenerate speed values
    - Player action deducts defaultActionCost; wait action deducts waitActionCost
    - Player-first tiebreak among actors with enough energy in the same tick
    - Interleaved turns: one action per actor per tick, no consecutive actions from speed alone
    - playerActionHandler and enemyActionHandler are injectable callbacks
    - Emits turn events via EventBus (TURN_START, TURN_END)
  </behavior>
  <action>
    1. Create src/engine/turn/turn-manager.ts:
       - Constructor takes: `world: World`, `eventBus: EventBus`, `config: TurnManagerConfig`
       - Private fields: `phase: TurnPhase` (starts AWAIT_INPUT), `playerActionHandler`, `enemyActionHandler`
       - `submitAction(action: string): void` — the main entry point:
         a. Guard: if phase !== AWAIT_INPUT, return silently (TURN-06)
         b. Set phase to PLAYER_ACTION
         c. Find player entity (query Actor where isPlayer === true)
         d. Deduct energy cost from player (defaultActionCost for move/attack, waitActionCost for wait)
         e. Call playerActionHandler(action, playerEntityId) — game layer handles the actual action
         f. Set phase to PRE_TURN, execute PRE_TURN systems via world.executeSystems(Phase.PRE_TURN)
         g. Set phase to ENEMY_TURNS, call processEnemyTurns()
         h. Set phase to POST_TURN, execute POST_TURN systems
         i. Flush event bus (eventBus.flush())
         j. Call advanceUntilPlayerReady()
         k. Set phase to AWAIT_INPUT
       - `processEnemyTurns(): void`:
         a. Get entities with Energy where energy.current >= threshold AND NOT isPlayer
         b. Sort by entity ID ascending (TURN-03)
         c. For each: check alive (Health.current > 0 or no Health component), skip dead (TURN-04)
         d. Call enemyActionHandler(entityId) — no-op in Phase 2
         e. Deduct defaultActionCost from entity's energy
       - `advanceUntilPlayerReady(): void`:
         a. While player's energy < threshold AND subTickCount < 100:
           - ENERGY_TICK: all living actors with Energy gain energy += speed
           - Process enemy turns (enemies that crossed threshold)
           - subTickCount++
         b. If subTickCount reaches 100, log error and break
       - `canAcceptInput(): boolean` — returns phase === AWAIT_INPUT
       - `setPlayerActionHandler(handler)` and `setEnemyActionHandler(handler)` setters
       - `start(): void` — initial energy tick to bring actors up to threshold, then await input
       - `getPhase(): TurnPhase`

    2. Write tests (RED then GREEN):
       - **Turn cycle order**: submitAction triggers player action → pre-turn → enemy turns → post-turn → await input (use spies on system execution to verify order)
       - **World frozen at AWAIT_INPUT**: canAcceptInput() returns true at AWAIT_INPUT, false during processing (TURN-02)
       - **Input gating**: submitAction is ignored when phase !== AWAIT_INPUT (TURN-06)
       - **Deterministic enemy order**: with 3 enemies (IDs 5, 2, 8), order is 2, 5, 8 (TURN-03)
       - **Player-first tiebreak**: player acts before any enemy in the same tick
       - **Dead entity skipping**: enemy with health.current <= 0 is skipped, no action handler called (TURN-04)
       - **Energy accumulation**: actor with speed 100 reaches threshold 1000 after 10 ticks (TURN-05)
       - **Fast enemy extra turns**: enemy with speed 200 acts twice per player turn (speed 100) — sub-tick loop processes enemy actions while player accumulates (TURN-05)
       - **Wait action**: wait deducts waitActionCost (500) instead of defaultActionCost (1000)
       - **Max sub-tick guard**: speed 1 player still eventually reaches threshold without infinite loop
       - **start() initializes energy**: after start(), if all actors have speed 100, all should reach threshold after initial ticks

    Tests should create a minimal World with mock components. Use world.createEntity(), addComponent() to set up actors. Spy on system execution order. Verify phase transitions and energy values after each step.
