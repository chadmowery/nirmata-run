**Module-level state:**
- `const entitySprites = new Map<EntityId, Sprite>()` — the strict 1:1 tracking map (where EntityId is `number`)

**createEntitySprite(entityId: EntityId, spriteKey: string, container: Container): Sprite**
- Guard: if `entitySprites.has(entityId)`, return existing sprite
- Create `new Sprite(Assets.get(spriteKey))` — texture from loaded spritesheet
- Set `sprite.width = TILE_SIZE; sprite.height = TILE_SIZE`
- `container.addChild(sprite)`
- `entitySprites.set(entityId, sprite)`

- `sprite.removeFromParent()` — detach from scene graph
- `sprite.destroy({ children: true })` — release GPU resources
- `entitySprites.delete(entityId)` — remove from tracking

**getEntitySprite(entityId: EntityId): Sprite | undefined** — map lookup
**getSpriteCount(): number** — map.size (for debug assertions)
**getEntitySprites(): ReadonlyMap<EntityId, Sprite>** — read-only access to full map

Create `src/rendering/animations.ts`:

**TweenAnimation interface:**
```typescript
interface TweenAnimation {
  entityId: EntityId;
  type: 'move' | 'attack-lunge' | 'attack-return' | 'death';
  startX: number;
  startY: number;
  targetX: number;
  targetY: number;
  elapsed: number;
  duration: number; // ms
  onComplete?: () => void;
}
```

**Module-level state:** `const activeAnimations: TweenAnimation[] = []`

**tickAnimations(deltaMs: number, getSprite: (id: EntityId) => Sprite | undefined): void**
- Iterate backwards (splice-safe removal)
- For each animation: advance elapsed by deltaMs, compute `t = min(elapsed / duration, 1)`
- Get sprite via getSprite(anim.entityId). If sprite gone, remove animation and continue.
- Movement: interpolate `sprite.x = startX + (targetX - startX) * t`, same for y
- Death: `sprite.alpha = 1 - t` (linear fade 1→0)
- Attack-lunge: interpolate position toward lunge target
- Attack-return: interpolate position back to original
- If `t >= 1`: call `onComplete?.()`, splice from array

**queueMoveTween(entityId: EntityId, fromX: number, fromY: number, toX: number, toY: number): void**
- Push movement animation: duration = 100ms, positions in pixel coords (gridX * TILE_SIZE)

**queueAttackAnimation(attackerId: EntityId, attackerPos: {x,y}, defenderPos: {x,y}, getSprite: (id: EntityId) => Sprite | undefined): void**
- Compute lunge target: 30% of the way from attacker to defender (pixel coords)
- Queue 'attack-lunge' (75ms) with onComplete that queues 'attack-return' (75ms back to original position)
- Apply red tint to defender: `defenderSprite.tint = 0xFF0000`. Use `setTimeout(() => { if (!sprite.destroyed) sprite.tint = 0xFFFFFF }, 150)` to reset.

**queueDeathAnimation(entityId: EntityId, onComplete: () => void): void**
- Queue 'death' animation: duration = 300ms, onComplete calls the cleanup callback (which should call destroyEntitySprite)

**hasActiveAnimation(entityId: EntityId): boolean** — check if any animation in array has matching entityId

**clearAnimations(): void** — for dungeon reset

  <files>src/rendering/render-system.ts, src/rendering/index.ts</files>
  <action>
Create `src/rendering/render-system.ts` — the ECS render system that orchestrates all rendering subsystems:

**createRenderSystem(config: RenderSystemConfig)** factory function:

```typescript
interface RenderSystemConfig {
  app: Application;
  layers: WorldLayers;
  world: World;
  eventBus: EventBus;
  getGrid: () => Grid;
  getPlayerEntity: () => EntityId;
  lightPasses: (x: number, y: number) => boolean;
}
```

Returns an object with:

**init(): void** — one-time setup:
- Subscribe to `ENTITY_CREATED`: if entity has Sprite component, call `createEntitySprite(entityId, spriteKey, layers.entityLayer)`
- Subscribe to `ENTITY_DESTROYED`: call `queueDeathAnimation(entityId, () => destroyEntitySprite(entityId))`
- Create `exploredSet = createExploredSet()`
- Add ticker callback: `app.ticker.add((ticker) => { tickAnimations(ticker.deltaMS, getEntitySprite); updateCameraFrame(ticker.deltaMS); })`

**renderSystem(world: World): void** — ECS system called each turn (or when FOV needs update):
- Get player position: `world.getComponent<Position>(getPlayerEntity(), 'position')`
- Compute FOV: `fovSet = computeFov(playerPos.x, playerPos.y, FOV_RADIUS, lightPasses, exploredSet)`
- Compute camera target: `target = computeCameraTarget(playerPos.x, playerPos.y)`
- Get visible range from camera: `visibleRange = getVisibleTileRange(layers.worldContainer.x, layers.worldContainer.y)`
- Rebuild tilemap with FOV + culling: `buildTilemap(getGrid(), layers.terrainLayer, { visibleRange, fovSet, exploredSet, playerPos })`
- Sync entity sprites:
  - For each entity with Position + Sprite: update sprite position (if not animating via hasActiveAnimation)
  - Apply entity visibility gating: `isEntityVisible(pos, entityType, fovSet, exploredSet)` → set `sprite.visible` and `sprite.alpha`
  - Handle items in itemLayer (visible on explored tiles, dimmed)

**Private updateCameraFrame(deltaMs: number):** called per frame by ticker:
- Get current camera target (stored from last renderSystem call)
- `const newPos = lerpCamera(worldContainer.x, worldContainer.y, targetX, targetY, deltaMs)`
- `layers.worldContainer.x = newPos.x; layers.worldContainer.y = newPos.y`

**onDungeonGenerated(): void** — called when a new dungeon is generated:
- `clearExplored(exploredSet)` — prevent phantom explored tiles from prior dungeon
- `clearAnimations()` — cancel any in-flight animations
- Do initial tilemap build
- Reset camera position (snap, don't lerp)

**destroy(): void** — cleanup:
- Unsubscribe event handlers
- Destroy all tracked sprites
- Remove ticker callbacks
