<objective>
Implement camera positioning, smooth lerp, and viewport culling so the camera centers on the player with responsive tracking and only visible tiles are submitted to the renderer.

Purpose: Without camera, the player can walk off-screen. Without culling, the full dungeon is rendered every FOV update (2000+ tiles vs ~600 visible). Camera lerp provides the responsive, direct-control feel specified by user decisions.
Output: Pure camera math functions ready to be wired into the render ticker and tilemap rebuild.
</objective>

  <behavior>
    - computeCameraTarget(15, 10) with 960×640 viewport and 32px tiles → target x = 960/2 - 15*32 - 16 = -16, y = 640/2 - 10*32 - 16 = -16 (centers player in viewport)
    - computeCameraTarget(0, 0) → target x = 960/2 - 0 - 16 = 464, y = 640/2 - 0 - 16 = 304
    - lerpCamera at t=0 (start) returns current position unchanged
    - lerpCamera after sufficient delta converges to within 1px of target
    - lerpCamera results are rounded to integer pixels (no sub-pixel jitter)
    - lerpCamera with deltaMs=1000 (1 second) reaches ~99.9% of target (fast ~50ms response)
  </behavior>
  <action>
Create `src/rendering/camera.ts` with pure math functions (no PixiJS dependency — operates on plain x/y numbers):

**computeCameraTarget(playerGridX: number, playerGridY: number): { x: number; y: number }**
- Computes the worldContainer offset that centers the player in the viewport
- Formula: `x = VIEWPORT_W / 2 - playerGridX * TILE_SIZE - TILE_SIZE / 2`
- Same for y with VIEWPORT_H
- Returns the TARGET position (not the lerped position)

**lerpCamera(currentX: number, currentY: number, targetX: number, targetY: number, deltaMs: number): { x: number; y: number }**
- Smooth exponential lerp with ~50ms response (tight, responsive feel per user decision)
- Lerp factor: `1 - Math.pow(0.001, deltaMs / 1000)` — reaches ~99.9% of target per second
- Apply: `newX = currentX + (targetX - currentX) * factor`
- **Round to integer pixels:** `Math.round(newX)` — prevents CompositeTilemap sub-pixel jitter (tile seams, blurry edges)
- Returns the new camera position

Tests verify centering math for various player positions, lerp convergence rate, and integer rounding.

    - getVisibleTileRange with camera offset → correct shifted range
    - Range always includes 2-tile padding on each side for smooth scrolling
    - Negative camera positions (scrolled right/down) produce positive tile ranges

  <action>
Add to `src/rendering/camera.ts`:

**getVisibleTileRange(cameraX: number, cameraY: number): { startX: number; startY: number; endX: number; endY: number }**
- Compute visible tile range from camera offset (worldContainer.x/y value)
- `startX = Math.floor(-cameraX / TILE_SIZE) - 1` (1-tile padding before)
- `startY = Math.floor(-cameraY / TILE_SIZE) - 1`
- `endX = startX + Math.ceil(VIEWPORT_W / TILE_SIZE) + 2` (1-tile padding after, +1 for partial tile)
- `endY = startY + Math.ceil(VIEWPORT_H / TILE_SIZE) + 2`
- This range is passed to `buildTilemap(grid, tilemap, { visibleRange })` to skip off-screen tiles

Tests verify correct tile range for various camera positions including edge cases: camera at origin, camera scrolled far right/down, camera at map corner.
