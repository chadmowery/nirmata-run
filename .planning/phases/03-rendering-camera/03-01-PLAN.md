**layers.ts:** Create world container factory:
- `createWorldContainer()` returns `{ worldContainer, terrainLayer, itemLayer, entityLayer, effectsLayer }`
- `worldContainer` = new Container()
- `terrainLayer` = new CompositeTilemap() — the batched GPU tilemap
- `itemLayer` = new Container() — item sprites
- `entityLayer` = new Container() — entity sprites
- `effectsLayer` = new Container() — animation effects
- Add children in ORDER: `worldContainer.addChild(terrainLayer, itemLayer, entityLayer, effectsLayer)` — this enforces render order terrain → items → entities → effects
- Export the return type as `WorldLayers` interface

**tilemap.ts:** CompositeTilemap build function (build-once pattern):
```typescript
interface BuildTilemapOptions {
  visibleRange?: { startX: number; startY: number; endX: number; endY: number };
  fovSet?: Set<string>;
  exploredSet?: Set<string>;
  playerPos?: { x: number; y: number };
}

function buildTilemap(
  grid: Grid,
  tilemap: CompositeTilemap,
  options?: BuildTilemapOptions
): void
```
- Call `tilemap.clear()` first
- Iterate grid tiles. If `visibleRange` provided, only iterate within range (viewport culling). Otherwise iterate all tiles.
- For each tile, compute alpha:
  - If `fovSet` provided and tile key in fovSet: use distance-based brightness gradient (1.0 at player, ~0.5 at FOV edge). Use `computeFovAlpha` helper (inline or imported).
  - If `exploredSet` provided and tile key in exploredSet: alpha = 0.3 (heavy dim per user decision)
  - If neither: alpha = 0.0 — skip tile (hidden = solid black = don't submit)
  - If NO fov/explored data provided (initial render): alpha = 1.0 for all tiles (pre-FOV mode)
- Map terrain type to spritesheet frame name: Floor→'floor', Wall→'wall', Door→'door'
- Call `tilemap.tile(textureName, x * TILE_SIZE, y * TILE_SIZE, { alpha })`
- Export `buildTilemap` and `tileTerrainToFrame` mapping
