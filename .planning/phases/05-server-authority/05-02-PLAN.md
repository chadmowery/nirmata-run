# Plan 05-02: Server Validation & API Route

## Goal
Implement the authoritative server-side logic and the Next.js API entry point for game sessions and action validation.

## Requirements
- **SRV-01**: Client sends action intents to Next.js API route via HTTP POST.
- **SRV-02**: Server validates all player actions (movement, attack, item pickup) against authoritative state.
- **SRV-03**: Server processes enemy turns during the player action request.
- **SRV-04**: Server returns full authoritative state snapshot (delta) after each action.

## Context
- **In-Memory Sessions**: Server-side singleton `Map<SessionId, { world, grid }>` stores the current state of each active game.
- **Single Round-Trip**: A single HTTP POST handles the player's intent AND all subsequent enemy turns.
- **Snap-to-Truth**: The server response is the absolute source of truth for the client.

## Tasks
1. **Scaffold API Route**:
   - Create `app/api/action/route.ts` with a `POST` handler.
   - Use `z.infer` types from `src/shared` for request/response validation.
2. **Implement Session Manager**:
   - Create a singleton `SessionManager` class to manage the `Map<SessionId, WorldState>`.
   - Provide methods for creating, retrieving, and updating sessions.
   - For v1, sessions are not persisted to a database (in-memory only).
3. **Integrate Action Pipeline**:
   - Within the API handler, retrieve the session's `World` and `Grid`.
   - Call `runActionPipeline` from `src/shared/pipeline.ts` with the player's `ActionIntent`.
   - Implement a server-side loop to process all AI turns until the player's next action is required.
   - Accumulate all changes (player + AI) into a single `StateDelta`.
4. **Integration Testing**:
   - Create `src/api/__tests__/action.test.ts`.
   - Use `vitest` and a mock request environment to verify the API correctly:
     - Rejects malformed requests (Zod validation).
     - Validates movement intents correctly.
     - Correctly processes enemy turns and returns the aggregated `StateDelta`.
     - Maintains session state between requests.

## Verification
- `npm run test src/api`: All API integration tests must pass.
- Verify using `curl` or a manual POST request that the API returns a valid `StateDelta` for a given session.
