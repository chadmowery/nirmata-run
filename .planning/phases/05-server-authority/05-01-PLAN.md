# Plan 05-01: Action Pipeline & Shared Types

## Goal
Establish the shared communication layer and the pure action processing function that enables both server-side validation and client-side optimistic prediction.

## Requirements
- **SRV-08**: Action pipeline is a pure function usable both client-side (local) and server-side (API route).
- **ARCH-04**: Shared types (action payloads, state snapshots) live in a shared/ directory used by both client and server.

## Context
- The project uses **Zod** for schema validation and **json-diff-ts** for calculating state deltas.
- **Serialization** is critical because the ECS `World` and `Grid` use `Map` and `Set`, which are not natively supported by `JSON.stringify`.
- The `ActionPipeline` must be "pure" to ensure deterministic results across environments (Node.js vs Browser).

## Tasks
1. **Setup Shared Directory**: Create `src/shared` and export Zod schemas for `ActionIntent` (discriminated union of MOVE, ATTACK, PICKUP, WAIT) and `StateDelta` (the output format of `json-diff-ts`).
2. **Implement Serialization Helpers**: 
   - Create `src/shared/serialization.ts`.
   - Implement `serializeWorld`/`deserializeWorld` to handle ECS `stores` (Maps) and `entities` (Sets/Arrays).
   - Implement `serializeGrid`/`deserializeGrid` to handle the grid's multi-layer `Map` data.
3. **Extract Action Logic**:
   - Create `src/shared/pipeline.ts`.
   - Implement `runActionPipeline(world: World, grid: Grid, action: ActionIntent): { world: World, grid: Grid, delta: StateDelta }`.
   - Wrap existing logic from `src/game/systems` (movement, combat, items) into this pipeline.
   - Ensure the pipeline returns a new `World`/`Grid` instance (or a patched one) and the calculated `delta`.
4. **Unit Testing**:
   - Add Vitest unit tests in `src/shared/__tests__/pipeline.test.ts`.
   - Verify that valid actions produce the expected `StateDelta`.
   - Verify that invalid actions (e.g., walking into a wall) produce an empty or "no-op" delta.
   - Verify that serialization/deserialization is lossless.

## Verification
- `npm run test src/shared`: All tests for pipeline and serialization must pass.
- Verify `src/shared` has zero imports from `src/game` or `src/rendering` to ensure environment neutrality.
