  <action>
    Create `src/game/systems/heat.ts` with `createHeatSystem()` factory function following the combat system pattern.

    Constructor parameters: `world: World<T>`, `eventBus: EventBus<T>`.

    Public methods:
    1. `dissipate(entityId)` — Called at start of player's turn. Reads `Heat` and `Shell` components. Calculates `effectiveDissipation = heat.baseDissipation + (shell.stability * 0.5)`. Reduces `heat.current` by that amount (minimum 0). If `heat.isVenting`, sets `isVenting = false`. Handles `heatPerTurn` for active toggle abilities: queries all Firmware entities in player's `FirmwareSlots.equipped`, checks each for `abilityDef.isActive && abilityDef.heatPerTurn > 0`, adds that Heat. Emits `HEAT_CHANGED` event.
    2. `addHeat(entityId, amount)` — Adds `amount` to `heat.current`. Emits `HEAT_CHANGED`.
    3. `vent(entityId)` — Removes `heat.ventPercentage` (50%) of current Heat. Sets `heat.isVenting = true`. Emits `VENT_COMPLETED` and `HEAT_CHANGED`. Emits `MESSAGE_EMITTED` with text "You vent excess Heat. Defense reduced until next turn."
    4. `isInCorruptionZone(entityId): boolean` — Returns `heat.current > heat.maxSafe`.
    5. `getHeatPercentage(entityId): number` — Returns `(heat.current / heat.maxSafe) * 100`.

    The `init()` method subscribes to `TURN_START` to call `dissipate()` on the player entity.
    The `dispose()` method unsubscribes.

    Add tests to `src/game/systems/heat.test.ts`:
    - dissipate() reduces Heat by baseDissipation + stability * 0.5
    - dissipate() clears isVenting flag
    - dissipate() does not reduce below 0
    - addHeat() increases current Heat accurately
    - vent() removes 50% of current Heat
    - vent() sets isVenting = true
    - isInCorruptionZone returns true when Heat > maxSafe
    - Stability modifier creates differentiated dissipation between Striker (stability 5) and Bastion (stability 10)
  </action>
