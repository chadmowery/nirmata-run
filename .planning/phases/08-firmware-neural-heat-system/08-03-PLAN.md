# Phase 8 - Plan 03: Kernel Panic System

<objective>
Create the KernelPanicSystem that detects overclocking (Heat above maxSafe), rolls against a 4-tier probability table modified by Shell Stability, and applies appropriate StatusEffect consequences. Integrate vent vulnerability into the combat damage pipeline.
</objective>

    Create `src/game/systems/kernel-panic.ts` with `createKernelPanicSystem()` factory function.

    Constructor parameters: `world: World<T>`, `eventBus: EventBus<T>`, `statusEffectSystem: StatusEffectSystem`.
    (Import `StatusEffectSystem` as `ReturnType<typeof createStatusEffectSystem>`.)

    Define the Kernel Panic tier table as a constant:
    ```typescript
    const KERNEL_PANIC_TABLE = [
      { minPercent: 101, maxPercent: 120, baseChance: 0.15, effectName: 'HUD_GLITCH', duration: 2, magnitude: 1, severity: 'minor' },
      { minPercent: 121, maxPercent: 140, baseChance: 0.30, effectName: 'INPUT_LAG', duration: 3, magnitude: 1, severity: 'moderate' },
      { minPercent: 141, maxPercent: 160, baseChance: 0.50, effectName: 'FIRMWARE_LOCK', duration: 2, magnitude: 1, severity: 'severe' },
      { minPercent: 161, maxPercent: Infinity, baseChance: 0.75, effectName: 'CRITICAL_REBOOT', duration: 3, magnitude: 1, severity: 'critical' },
    ] as const;
    ```

    Public methods:
    1. `checkOverclock(entityId: EntityId): KernelPanicResult | null`
       - Read `Heat` and `Shell` components from entity.
       - Calculate Heat percentage: `heatPercent = (heat.current / heat.maxSafe) * 100`.
       - If `heatPercent <= 100`, return null (no overclock).
       - Find matching tier from `KERNEL_PANIC_TABLE` where `heatPercent >= tier.minPercent && heatPercent <= tier.maxPercent`.
       - Calculate effective chance: `effectiveChance = Math.max(0, tier.baseChance - (shell.stability * 0.01))`. This means stability=5 reduces chance by 5%, stability=15 reduces by 15%.
       - Roll: `Math.random() < effectiveChance`.
       - If roll succeeds:
         * Apply status effect via `statusEffectSystem.applyEffect(entityId, { name: tier.effectName, duration: tier.duration, magnitude: tier.magnitude, source: 'kernel_panic' })`.
         * If `CRITICAL_REBOOT`: additionally force `heat.current = 0` (self-correcting — D-11). The stun (skip N turns) is handled by the TurnManager checking for CRITICAL_REBOOT status.
         * Emit `KERNEL_PANIC_TRIGGERED` event with tier number, effectName, severity.
         * Emit `MESSAGE_EMITTED` with severity-appropriate text:
           - minor: `"⚠ KERNEL_PANIC_DETECTED: HUD_GLITCH — Display artifacts for 2 turns"`
           - moderate: `"⚠ KERNEL_PANIC_DETECTED: INPUT_LAG — Controls sluggish for 3 turns"`
           - severe: `"⚠⚠ KERNEL_PANIC_CRITICAL: FIRMWARE_LOCK — Firmware offline for 2 turns"`
           - critical: `"⚠⚠⚠ CRITICAL_REBOOT — System rebooting. Stunned for 3 turns. Heat vented to 0."`
         * Return `{ tier: tierIndex, effectName, rolled: true, effectApplied: true }`.
       - If roll fails: return `{ tier: tierIndex, effectName, rolled: true, effectApplied: false }`.

    2. `getTier(heatPercent: number): typeof KERNEL_PANIC_TABLE[number] | null`
       - Returns the matching tier or null if not overclocking.

    The `init()` method subscribes to `FIRMWARE_ACTIVATED` event. When received, calls `checkOverclock(payload.entityId)` — this implements D-10 (Kernel Panic check AFTER ability resolves).
    The `dispose()` method unsubscribes.

    Export type: `export type KernelPanicSystem = ReturnType<typeof createKernelPanicSystem>;`
    Export result type: `export interface KernelPanicResult { tier: number; effectName: string; rolled: boolean; effectApplied: boolean; }`

    Create `src/game/systems/kernel-panic.test.ts` with tests:
    - checkOverclock returns null when Heat <= maxSafe (Heat=80, maxSafe=100)
    - checkOverclock selects tier 1 (HUD_GLITCH) for Heat 110% of maxSafe
    - checkOverclock selects tier 2 (INPUT_LAG) for Heat 130% of maxSafe
    - checkOverclock selects tier 3 (FIRMWARE_LOCK) for Heat 150% of maxSafe
    - checkOverclock selects tier 4 (CRITICAL_REBOOT) for Heat 170% of maxSafe
    - Stability reduces effective chance (mock Math.random to verify threshold)
    - High stability can reduce chance to 0 (guaranteed safe at tier 1 with stability >= 15)
    - CRITICAL_REBOOT forces Heat to 0
    - CRITICAL_REBOOT applies status effect with duration 3
    - checkOverclock emits KERNEL_PANIC_TRIGGERED event when roll succeeds
    - checkOverclock does NOT apply effect when roll fails
