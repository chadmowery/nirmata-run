  <action>
    Create `src/game/systems/firmware.ts` with `createFirmwareSystem()` factory function.

    Constructor parameters: `world: World<T>`, `grid: Grid`, `eventBus: EventBus<T>`, `movementSystem: MovementSystem`, `heatSystem: HeatSystem`.
    (Import `HeatSystem` as `ReturnType<typeof createHeatSystem>`.)

    Public methods:
    1. `activateAbility(entityId: EntityId, slotIndex: number, targetX: number, targetY: number): boolean`
       - Read `FirmwareSlots` from entity. Validate `slotIndex` < `equipped.length` and `equipped[slotIndex]` is a valid entity.
       - Read `AbilityDef` from the firmware entity at `equipped[slotIndex]`.
       - If `abilityDef.isLegacy`, double the effective `heatCost` (FIRM-11 stub).
       - Call `heatSystem.addHeat(entityId, effectiveHeatCost)`.
       - Resolve effect based on `abilityDef.effectType`:
         * `'dash'`: Move entity from current position to (targetX, targetY) directly. Validate target is within `abilityDef.dashDistance` tiles (Manhattan distance). Update `Position` component, call `grid.moveEntity()`. Emits `ENTITY_MOVED`. Ignores collision (Phase_Shift passes through walls/entities).
         * `'ranged_attack'`: Validate target within `abilityDef.range`. Check for entities at (targetX, targetY) using `grid.getEntitiesAt()`. For each hostile entity found, read their `Health` and `Defense` components, calculate damage = `abilityDef.damageAmount - defense.armor` (min 1), apply to health. Emit `DAMAGE_DEALT`. If health <= 0, emit `ENTITY_DIED`, destroy entity, remove from grid.
         * `'toggle_vision'`: Toggle `abilityDef.isActive`. If activating, Heat is added by the per-turn hook in HeatSystem. If deactivating, no Heat cost. Emit `FIRMWARE_TOGGLED`.
       - Emit `FIRMWARE_ACTIVATED` event with all parameters.
       - Emit `MESSAGE_EMITTED` with ability name and effect: e.g., "Phase_Shift.sh activated! Dashed to (x, y)."
       - Return true on success, false if validation fails.

    2. `getAbilityDef(entityId: EntityId, slotIndex: number): AbilityDefData | null`
       - Convenience getter for UI/targeting system to read ability parameters.

    The `init()` method is a no-op (system is called imperatively, not event-driven).
    The `dispose()` method is a no-op.

    Create `src/game/systems/firmware.test.ts` with tests:
    - activateAbility returns false when no firmware equipped at slot
    - activateAbility adds heatCost to player's Heat
    - activateAbility with dash effectType moves player to target position
    - activateAbility with ranged_attack effectType deals damage to target entity
    - activateAbility with toggle_vision toggles isActive on ability entity
    - activateAbility with isLegacy=true doubles the Heat cost
    - activateAbility emits FIRMWARE_ACTIVATED event
    - activateAbility rejects target out of range
  </action>

    `TargetingManager` public methods:
    1. `startTargeting(slotIndex: number, playerX: number, playerY: number, range: number, effectType: string)` — Set `active=true`, `cursorX/Y = playerX/Y`, store parameters. Emit `TARGETING_STARTED`.

    4. `cancel()` — Set `active=false`. Emit `TARGETING_CANCELLED`.
    5. `isActive(): boolean` — Returns `active`.
    6. `getState(): TargetingState` — Returns current state.
