<task type="auto" tdd="true">
  <name>Task 2: AI decision-making system with A* pathfinding</name>
  <files>src/game/systems/ai.ts, src/game/systems/ai.test.ts</files>
  <behavior>
    - AI system provides a `processEnemyTurn(entityId)` function for the turn manager
    - Step 1: Compute enemy's FOV from their position using rot-js PreciseShadowcasting
    - Step 2: Check if player tile is in enemy's FOV → update FovAwareness component
    - Step 3: State transitions based on awareness:
      - idle → chasing: when canSeePlayer becomes true
      - chasing → attacking: when adjacent to player (Manhattan distance ≤ 1)
      - chasing → idle: when can't see player AND no lastKnownPlayerX (lost target)
      - attacking → chasing: when not adjacent to player
      - attacking → idle: when can't see player
    - Step 4: Execute behavior:
      - idle: do nothing
      - chasing: compute A* path from enemy to player, move one step along path
      - attacking: move toward player (dx, dy toward player), triggering bump-attack via movement system
    - A* uses rot-js AStar with passableCallback checking grid.isWalkable
  </behavior>
  <action>
    1. Create src/game/systems/ai.ts:
       ```typescript
       import AStar from 'rot-js/lib/path/astar';
       import PreciseShadowcasting from 'rot-js/lib/fov/precise-shadowcasting';
       
       export function createAISystem(
         world: World,
         grid: Grid,
         movementSystem: MovementSystem,
         eventBus: EventBus<GameEvents>
       ): AISystem
       ```
       - `processEnemyTurn(entityId: EntityId): void` — the main entry point
       - Internal `computeEnemyFov(enemyX, enemyY, radius)`:
         - Use PreciseShadowcasting with `(x, y) => grid.isTransparent(x, y)`
         - Return Set<string> of visible tile keys
       - Internal `findPlayerEntity()`: query world for Actor where isPlayer === true
       - Internal `isAdjacent(pos1, pos2)`: Manhattan distance ≤ 1, not same position
       - Internal `pathfindToward(fromX, fromY, toX, toY)`:
         - Create AStar with `passableCallback: (x, y) => grid.isWalkable(x, y)`
         - Compute path, return first step (dx, dy) or null if no path
       - State machine logic in processEnemyTurn:
         a. Get enemy's Position, AIState, FovAwareness
         b. Compute enemy FOV, check if player pos is visible
         c. Update FovAwareness.canSeePlayer + lastKnownPlayerX/Y
         d. Run state transitions
         e. Execute: idle=noop, chasing=pathfind+move, attacking=bump-attack

    2. Create src/game/systems/ai.test.ts:
       - Setup: create World, Grid (10x10), EventBus, MovementSystem, AISystem
       - Create player entity at (5, 5) with Actor { isPlayer: true }

       - **FOV awareness tests:**

       - **State transition tests:**
