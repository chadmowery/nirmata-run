  <name>Task 1: Engine generation interface and BSP tree algorithm</name>
  <files>src/engine/generation/types.ts, src/engine/generation/bsp.ts, src/engine/generation/bsp.test.ts</files>
  <behavior>
    - GeneratorConfig defines: width, height, minRoomSize (default 5), maxRoomSize (default 12), minRooms (default 5), maxSplitDepth (default 8), corridorWidth (default 1), seed (string)
    - Room is { x, y, width, height } representing the interior floor area
    - BSP recursively splits the grid space into leaves, carves rooms inside leaves, connects siblings with corridors
    - Same seed always produces identical room layout
    - Rooms have minimum 3×3 interior (minRoomSize includes walls, so interior = minRoomSize - 2)
    - Corridors connect room centers horizontally then vertically (L-shaped)
    - Doors placed where corridors meet room edges
  </behavior>
  <action>
    1. Create src/engine/generation/types.ts:
       ```typescript
       export interface GeneratorConfig {
         width: number;
         height: number;
         seed: string;
         minRoomSize?: number;    // default 5 (interior 3×3)
         maxRoomSize?: number;    // default 12
         minRooms?: number;       // default 5
         maxSplitDepth?: number;  // default 8
         corridorWidth?: number;  // default 1
       }

       export interface Room {
         x: number;      // top-left x of interior
         y: number;      // top-left y of interior
         width: number;  // interior width
         height: number; // interior height
         centerX: number;
         centerY: number;
       }

       export interface Corridor {
         points: Array<{ x: number; y: number }>;
       }

       export interface GeneratorResult {
         rooms: Room[];
         corridors: Corridor[];
         doors: Array<{ x: number; y: number }>;
       }
       ```

    2. Create src/engine/generation/bsp.ts:
       - Import `RNG` from rot-js (use `import RNG from 'rot-js/lib/rng'` or `rot-js/lib/rng/alea`)
       - `export function generateBSP(config: GeneratorConfig): GeneratorResult`
       - Internal BSP node: `{ x, y, w, h, left?, right?, room? }`
       - Algorithm:
         a. Seed RNG: `RNG.setSeed(config.seed)` (or create Alea instance)
         b. Create root node spanning entire grid (minus 1-tile border)
         c. Recursively split leaves:
            - Choose split direction based on aspect ratio (split along longer axis)
            - Split position: random within 40%-60% of dimension (prevents extreme splits)
            - Stop splitting if: node too small for 2 rooms, or depth >= maxSplitDepth
         d. Carve rooms in leaf nodes:
            - Room interior size: random between minRoomSize-2 and min(maxRoomSize-2, leaf dimension - 2)
            - Position room randomly within leaf with at least 1-tile padding
            - Record Room with center coordinates
         e. Connect sibling rooms with L-shaped corridors:
            - Walk up the BSP tree, connecting pairs
            - Find nearest rooms in left/right subtrees
            - Carve horizontal then vertical corridor between room centers
         f. Generate door positions:
            - Where corridor enters a room (transition from corridor to room floor)
       - Return GeneratorResult with all rooms, corridors, and door positions

    3. Create src/engine/generation/bsp.test.ts:
       - **Seed reproducibility:**
         - Same seed → identical rooms (position, size)
         - Different seed → different rooms
       - **Room constraints:**
         - All rooms have interior >= 3×3
         - All rooms fit within grid bounds
         - Room count >= minRooms (re-split if needed or validate)
         - No rooms overlap
       - **Corridor tests:**
         - Every pair of siblings is connected
         - Corridors stay within grid bounds
       - **Configuration tests:**
         - Custom minRoomSize respected
         - Custom maxRoomSize respected
         - Different grid sizes produce valid results
       - **Edge cases:**
         - Very small grid (20×20) still generates at least one room
         - Very large grid (100×100) generates multiple rooms
  </action>

  </verify>

       export interface ConnectivityResult {
         connected: boolean;
         reachable: number;
         total: number;
         unreachableRegions: Array<{ x: number; y: number }>;
       }

       export function validateConnectivity(
         width: number,
         height: number,
         isWalkable: (x: number, y: number) => boolean,
         startX: number,

       ): ConnectivityResult

       - After flood fill: iterate entire grid counting total walkable tiles

  <files>src/game/generation/dungeon-generator.ts, src/game/generation/dungeon-generator.test.ts</files>
  <behavior>
    - generateDungeon takes config, creates Grid, runs BSP, writes tiles, validates connectivity
    - All tiles start as walls (walkable: false, transparent: false)
    - Room interiors carved as floors (walkable: true, transparent: true)
    - Corridors carved as floors
    - Doors carved as walkable + transparent with terrain 'door'

    - Returns { grid, rooms, doors, playerSpawnRoom }
  </behavior>
  <action>
    1. Create src/game/generation/dungeon-generator.ts:
       ```typescript
       export interface DungeonConfig extends GeneratorConfig {
         // Inherits width, height, seed, room params
       }

       export interface DungeonResult {
         grid: Grid;
         rooms: Room[];
         doors: Array<{ x: number; y: number }>;
         playerSpawnRoom: Room;
       }

       export function generateDungeon(config: DungeonConfig): DungeonResult
       ```
       - Create Grid(config.width, config.height)
       - Set ALL tiles to wall: terrain='wall', walkable=false, transparent=false
       - Run generateBSP(config) → rooms, corridors, doors
       - Carve rooms: for each room, setTile to floor for interior tiles
       - Carve corridors: for each corridor point, setTile to floor
       - Carve doors: setTile with terrain='door', walkable=true, transparent=true
       - Select player spawn room: first room (deterministic from seed)
       - Validate connectivity: flood fill from spawn room center
       - If not connected: retry with modified seed (append attempt number), max 10 attempts
       - If all attempts fail: throw Error('Failed to generate connected dungeon')
       - Return DungeonResult
