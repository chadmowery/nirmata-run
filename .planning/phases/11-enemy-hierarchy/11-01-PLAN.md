2. **Null-Pointer handler** (inside ai.ts per D-07):
   - `processNullPointerTurn(entityId)`:
     - Get entity position, compute FOV, check if player visible
     - If player visible AND Manhattan distance <= 8: teleport behind player
       - "Behind player" algorithm: calculate direction vector from entity to player, reverse it. Target tile = player position + reversed direction * 1 (the tile directly opposite the enemy relative to the player). If that tile is not walkable, try adjacent tiles around that target. If none walkable within 3-tile radius, don't teleport and use basic chase instead.
       - On teleport: update Position component, call `grid.removeEntity(entityId, oldX, oldY)` then `grid.addEntity(entityId, newX, newY)`, emit `ENEMY_TELEPORTED` event
     - If player visible AND adjacent: use bump attack via `movementSystem.processMove(entityId, dx, dy)` toward player. On successful hit, apply HUD_GLITCH status effect for 2 turns (magnitude 1) via pushing to StatusEffects.effects array on the player entity
     - If player not visible: fall back to base CHASING behavior (reuse existing pathfinding logic)
   - Stats (discretionary): sightRadius=8, energy speed=120 (per D-03 — fast)

3. **Buffer-Overflow handler** (inside ai.ts per D-04/D-05):
   - `processBufferOverflowTurn(entityId)`:
     - Get PackMember component. If leader: pathfind toward player normally
     - If follower: calculate surround position. Get leader position, get player position. Target = one of the 4 cardinal tiles around the player that is closest to this follower. Pathfind toward that target tile.
     - After movement, the pack-coordinator system handles detonation check
   - Stats (discretionary): sightRadius=6, energy speed=80 (per D-03 — moderate)

4. **Create pack-coordinator system** (`src/game/systems/pack-coordinator.ts`):
   - Factory function: `createPackCoordinatorSystem<T extends GameplayEvents>(world: World<T>, grid: Grid, eventBus: EventBus<T>)`
   - Track which packIds have detonated this turn (Set<string>) to prevent multi-detonation per turn (per Pitfall 1 from research)
   - Listen to `ENTITY_MOVED` events: check if mover has PackMember component. If so, count how many members of that pack are adjacent (Manhattan distance <= 1) to the player. If count >= 3 (per D-05), trigger detonation for all adjacent members.
   - Detonation: emit `PACK_DETONATION` event per member (damage=8, AOE), apply `MOVEMENT_SLOW` status effect (duration=2, magnitude=50) to player via StatusEffects, emit `MESSAGE_EMITTED` with "Buffer-Overflow pack detonated!" text, destroy the detonating entities via `world.destroyEntity()`
   - Listen to turn-end event (or expose a `resetTurnState()` method) to clear the detonated-this-turn set
   - Return `{ init(), dispose(), resetTurnState() }`
