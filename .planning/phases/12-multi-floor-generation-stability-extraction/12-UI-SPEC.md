# Phase 12 — UI Design Contract

> Visual and interaction contract for System Handshake UI, BSOD screen, Run Results screen, Stability bar, and Floor indicator. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|

| Component library | none (custom React components) |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.5 |
| Heading | 24px | 600 | 1.2 |
| Display | 48px | 600 | 1.0 |

**Source:** Body/label sizes match existing HUD text readability (styles.module.css .messageEntry at 0.85rem ≈ 13.6px, rounded to 14px). Heading size appropriate for overlay titles. Display size for BSOD/Anchor titles (VIS-02 "large, condensed sans-serif headers"). Two weights only (regular + semibold) for simplicity. Line heights follow standard web typography (1.5 for body, 1.2 for headings, 1.0 for display to avoid excessive height on large text).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #000000 | Background, world desaturation base |
| Secondary (30%) | rgba(0, 0, 0, 0.90) | Overlay backgrounds (menuOverlay pattern) |
| Accent Cyan (10%) | #00F0FF | Extract button, Stability bar (safe zone), "Secure" actions |
| Accent Pink (10%) | #FF0055 | Descend button, Stability bar (danger zone), "Risk" actions |
| Semantic: Success | #00F0FF | Extraction success state, positive stat indicators |
| Semantic: Danger | #FF0055 | BSOD screen, death state, Stability critical |
| Semantic: Warning | #FF9500 | BSOD accent color (Safety Orange from VIS-06) |

Accent reserved for:
- Cyan: Extract button, "DE-REZZ & EXTRACT" label, Stability bar above 30%, extraction results heading
- Pink: Descend button, "STABILIZE & DESCEND" label, Stability bar below 30%, BSOD screen primary color, death results heading

**Source:** VIS-01 color palette (#000000, #FFFFFF, #00F0FF, #FF0055). VIS-06 specifies Safety Orange for BSOD. VIS-07 specifies cyan/pink decision blocks. CONTEXT.md D-18 mandates exact color palette in Phase 12. Existing globals.css uses green terminal aesthetic (#00ff41), but Phase 12 introduces the final "Vibrant Decay" palette per REQUIREMENTS.md.

## Interaction States

### AnchorOverlay

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Mount | World desaturates to grayscale (PixiJS ColorMatrixFilter), overlay fades in 200ms, game pauses | Player steps on Anchor entity |
| Extract hover | Cyan button glow (box-shadow: 0 0 20px #00F0FF) | Mouse over Extract button |
| Descend hover | Pink button glow (box-shadow: 0 0 20px #FF0055) | Mouse over Descend button |
| Insufficient Scrap | Descend button disabled, error message below button, pink text | ScrapComponent.amount < descendCost |
| Unmount (Extract) | Overlay fades out 200ms, world recolorizes, de-rezz animation 1-2s, transition to RunResultsScreen | Extract button clicked |
| Unmount (Descend) | Overlay fades out 200ms, world recolorizes, floor transition glitch 600ms, game resumes | Descend button clicked |

### StabilityBar

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Safe (31-100%) | Cyan bar fill, smooth gradient, no effects | Stability.current > 30 |
| Warning (11-30%) | Pink bar fill, scanline overlay flicker | Stability.current 11-30 |
| Critical (1-10%) | Pink bar fill, rapid pulse animation, HUD scanlines intensify | Stability.current 1-10 |
| Degraded (0%) | Bar empty, HUD corruption (text glitch, color inversion per D-12), HP bleed notification | Stability.current === 0 |

### BSODScreen

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Display | Full-screen Safety Orange (#FF9500), FATAL_EXCEPTION heading, context reason, static 2-3 second display | Player death event |
| Auto-transition | Fade to black 300ms, transition to RunResultsScreen (death variant) | After 2-3 second display |

### FloorIndicator

| State | Visual Treatment | Trigger |
|-------|------------------|---------|
| Floor transition | Flash animation 500ms (opacity pulse), depth band label updates | FLOOR_TRANSITION event |
| Normal | Static display, no animation | All other states |

**Source:** CONTEXT.md D-07 (floor transition timing), D-12 (degraded state visuals), D-17 (overlay pause), D-18 (System Handshake visuals), D-32 (BSOD timing). VIS-07 (desaturation transition). Interaction states follow existing hover pattern from styles.module.css .menuButton:hover.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Tab order: Extract button → Descend button. Enter/Space activates. Escape closes overlay (returns to game if enemies present — dangerous per D-16). |
| Focus indicators | 2px cyan outline on focused button (outline: 2px solid #00F0FF) |
| Screen reader labels | aria-label on all buttons, aria-live="polite" on Stability bar for percentage announcements, role="dialog" on AnchorOverlay with aria-labelledby |
| Color contrast | All text on black background meets WCAG AAA (21:1 for white, 14:1 for cyan, 10:1 for pink — all exceed 7:1 AAA threshold) |
| Animation opt-out | Respect prefers-reduced-motion: disable glitch/fade transitions, instant overlay mount, static bar fills |

**Source:** Standard WCAG 2.1 AAA compliance. Existing project does not implement accessibility features yet (no aria labels in GameOver.tsx), but Phase 12 establishes pattern.

## Animation Timing

| Animation | Duration | Easing | Trigger |
|-----------|----------|--------|---------|
| Overlay fade in | 200ms | ease-out | AnchorOverlay mount |
| Overlay fade out | 200ms | ease-out | AnchorOverlay unmount |
| World desaturation | 200ms | linear | ColorMatrixFilter apply |
| World recolorization | 200ms | linear | ColorMatrixFilter remove |
| Floor transition glitch | 200ms | none (static) | FLOOR_TRANSITION event |
| Floor transition fade out | 200ms | ease-in | After glitch |
| Floor transition fade in | 200ms | ease-out | After generation |
| Floor number flash | 500ms | ease-in-out | After fade in |
| Stability bar fill change | 300ms | ease-out | STABILITY_CHANGED event |
| Button hover glow | 200ms | ease-out | Mouse enter |
| BSOD display | 2500ms | none (static) | Death event |
| BSOD fade out | 300ms | ease-in | Auto-transition to results |
| De-rezz extraction | 1500ms | ease-in | Extract action confirmed |

**Source:** CONTEXT.md D-07 (floor transition timing: 200ms + 200ms + 200ms = 600ms total), D-21 (de-rezz 1-2s, use 1.5s midpoint), D-32 (BSOD 2-3s, use 2.5s midpoint). Standard web UI timing (200-300ms for transitions). Existing styles.module.css .barFill uses 300ms ease-out.
