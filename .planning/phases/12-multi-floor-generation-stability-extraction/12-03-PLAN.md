**1. Create screen-effects.ts** (`src/rendering/filters/screen-effects.ts`):
Per RESEARCH.md Pattern 5, create PixiJS filter helpers for world desaturation:

```typescript
import { ColorMatrixFilter, Container } from 'pixi.js';

// Cache a single filter instance to avoid memory leaks (per pitfall 4)
let _grayscaleFilter: ColorMatrixFilter | null = null;
let _desatFilter: ColorMatrixFilter | null = null;

export function createDesaturationFilter(): ColorMatrixFilter {
  const filter = new ColorMatrixFilter();
  filter.desaturate();
  return filter;
}

export function applyGrayscaleToContainer(container: Container): void {
  // Reuse single instance per pitfall 4
  if (!_grayscaleFilter) {
    _grayscaleFilter = createDesaturationFilter();
  }
  container.filters = [_grayscaleFilter];
}

export function removeFiltersFromContainer(container: Container): void {
  container.filters = null;
}

export function applyStabilityDesaturation(
  container: Container,
  stabilityPercent: number,
): void {
  if (stabilityPercent > 50) {
    // No desaturation above 50%
    if (container.filters) {
      container.filters = null;
    }
    return;
  }
  // 50% stability = 0% desaturation, 0% stability = 100% desaturation
  if (!_desatFilter) {
    _desatFilter = new ColorMatrixFilter();
  }
  _desatFilter.reset();
  _desatFilter.desaturate();
  // Scale by how far below 50% we are
  _desatFilter.alpha = (50 - stabilityPercent) / 50;
  container.filters = [_desatFilter];
}

export function disposeScreenEffects(): void {
  if (_grayscaleFilter) {
    _grayscaleFilter.destroy();
    _grayscaleFilter = null;
  }
  if (_desatFilter) {
    _desatFilter.destroy();
    _desatFilter = null;
  }
}
```

**1. Create AnchorInteractionSystem** (`src/game/systems/anchor-interaction.ts`):
Per D-15 through D-23, this system handles the full Anchor interaction flow.

Factory: `export function createAnchorInteractionSystem(world, grid, eventBus, turnManager, playerId)` returning `{ init, dispose }`.

The system must:

**On player movement (listen for ENTITY_MOVED or equivalent):**
- After player moves, check if the destination tile has an entity with `AnchorMarker` component
- If yes AND `AnchorMarker.used === false`:
  1. Call `turnManager.pause()` — stops all turn processing (per D-16)
  2. Emit `APPLY_WORLD_FILTER` event with `{ filterType: 'grayscale' }` — triggers render system desaturation
  3. Gather overlay data:
     - `floorNumber`: from player's `FloorState.currentFloor`
     - `stabilityPercent`: from player's `Stability.current`
     - `inventory`: read player's FirmwareSlots, AugmentSlots, SoftwareSlots, BurnedSoftware — collect item names and rarities into categorized lists
     - `scrap`: from player's `Scrap.amount`
     - `descendCost`: calculate as `50 + (floorNumber * 10)` — base 50 Scrap, +10 per floor (per D-23, Claude's discretion)
     - `nextFloorEnemyTier`: derive from depth-config.json bands — find band for `floorNumber + 1`, use its label
     - `estimatedStabilityAfterDescent`: current stability minus floor chunk drain for next floor, plus 50% refill
  4. Call `gameStore.getState().showAnchorOverlay(overlayData)` — mounts the React overlay

**On ANCHOR_DECISION event (emitted by React overlay):**
The React overlay will emit an event or call a callback when the player makes a decision.
Listen for a custom event `ANCHOR_DECISION_MADE` with payload `{ decision: 'extract' | 'descend', anchorId: number }`:

- If `decision === 'extract'`:
  1. Dispatch `ANCHOR_EXTRACT` action through pipeline (calls pipeline handler from 12-02)
  2. Emit `REMOVE_WORLD_FILTER` event
  3. Call `turnManager.resume()`
  4. Call `gameStore.getState().hideAnchorOverlay()`
  5. Show run results (extraction variant) via `gameStore.getState().showRunResults(...)`

- If `decision === 'descend'`:
  1. Read descendCost, dispatch `ANCHOR_DESCEND` action through pipeline with `{ cost: descendCost, anchorId }`
  2. If pipeline succeeds (no INSUFFICIENT_SCRAP error):
     - Emit `REMOVE_WORLD_FILTER` event
     - Call `turnManager.resume()`
     - Call `gameStore.getState().hideAnchorOverlay()`
     - Emit `STAIRCASE_INTERACTION` event to trigger floor manager's `descendToFloor` (the Anchor doubles as descent trigger after Descend decision)
  3. If insufficient scrap, the pipeline emits an error message — overlay stays open, player can choose Extract instead

**Also handle staircase step-on (per D-04):**
Listen for player movement to staircase tiles (entity with `StaircaseMarker`):
- Set a flag in the UI store: `showDescentConfirmation: true` with `targetFloor`
- On confirmation (user clicks "yes" in a simple dialog):
  - Emit `STAIRCASE_INTERACTION` event with `{ entityId: playerId, staircaseId, targetFloor }`
  - The floor-manager system (from 12-01) handles the actual descent
