**2. Extend `placeEntities()` in entity-placement.ts** to support:
- **Staircase placement (per D-03):** After placing enemies/items, find the room with the greatest graph distance from playerSpawnRoom (use room center-to-center Euclidean distance as proxy). Place a staircase entity at that room's center with `Position`, `SpriteComponent` (char: '>', fg: '#FFFFFF'), `StaircaseMarker` (targetFloor: depth + 1). Do NOT place staircase on the last floor (depth >= maxFloor from depth-config.json).
- **Anchor placement (per D-15):** On floors 5, 10, 15 (i.e., when `depth % 5 === 0 && depth > 0`), place an Anchor entity in a random non-spawn room with `Position`, `SpriteComponent` (char: 'A', fg: '#00F0FF'), `AnchorMarker` (used: false). The Anchor room MUST NOT be the staircase room or spawn room.
- **Special room types (per D-25):** After selecting rooms for staircase/anchor/spawn, tag remaining rooms as candidates for special types. Roll against `specialRoomProbability` from depth-config.json for each candidate room:
  - Treasure rooms: spawn 2-3 loot items from loot-distribution.json with no enemies
  - Challenge rooms: spawn 50% more enemies than normal, mark with a `challengeRoom` flag
  - Hazard rooms: mark tiles in room with a hazard flag (implementation TBD, just tag the room for now)
  - Use depth-appropriate loot from loot-distribution.json for treasure room items
- **Loot drops (per D-26, FLOOR-05):** When placing items, use `loot-distribution.json` tables. Select loot template by weighted random from the matched depth band. `firmware-drop` and `augment-drop` template names are placeholders — create the entity with `Item` component and a `name` field matching the template string. Full firmware/augment entity creation depends on Phase 13 compilation; for now these are inventory tokens.

**3. Create `createFloorManagerSystem()`** in `src/game/systems/floor-manager.ts`:
Follow the existing system factory pattern (`createXSystem()` returning `{ init(), dispose() }`).

The system manages floor transitions per D-01:
- Export `createFloorManagerSystem(world, grid, eventBus, entityFactory, componentRegistry, playerId)` returning `{ init, dispose, descendToFloor }`
- `descendToFloor(newFloor: number, runSeed: string)`:
  1. Snapshot all entity IDs: `const entities = [...world.getAllEntities()]` (per pitfall 2 — snapshot before iteration)
  2. Destroy all non-player entities: `entities.filter(id => id !== playerId).forEach(id => world.destroyEntity(id))`
  3. Clear grid: `grid.clear()`
  4. Generate floor seed: `const floorSeed = \`${runSeed}_floor_${newFloor}\`` (per pitfall 1 — underscore delimiter)
  5. Hash seed: reuse `hashSeedForPlacement()` pattern from engine-factory.ts
  6. Call `RNG.setSeed(hash)` from rot-js
  7. Call `generateDungeon({ width: grid.width, height: grid.height, seed: floorSeed, depth: newFloor })`
  8. Replace grid terrain from dungeon result
  9. Move player to new spawn room center (update Position component, update grid entity tracking)
  10. Call `placeEntities()` with depth: newFloor for enemies, loot, staircase, anchor
  11. Update `FloorState` component on player entity: set `currentFloor = newFloor`
  12. Emit `FLOOR_TRANSITION` event: `{ floorNumber: newFloor, depthBand: getDepthBand(newFloor).label }`
- `init()`: Listen for `STAIRCASE_INTERACTION` event. When received, call `descendToFloor(targetFloor, runSeed)` where runSeed comes from player's FloorState component.

- Pass `depth: 1` to the initial `placeEntities()` call (first floor always depth 1)
